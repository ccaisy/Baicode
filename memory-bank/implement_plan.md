# 实施计划：CLI AI Coding Agent (MVP 版本)

## 阶段一：CLI 基建与网关打通 (Milestone 1)

这一阶段的核心是搭建终端骨架，并确保大模型通信链路无阻。

### Step 1: 环境初始化与配置加载

- **指令**：创建 Python 虚拟环境。引入 `python-dotenv` 库，建立环境变量管理模块。定义一个专门负责读取项目根目录下 `.env` 文件中各个模型 API Key（如 OpenAI、DeepSeek）的加载函数。
- **测试**：编写一个测试脚本，断言该加载函数能够成功从本地环境读取并返回 `.env` 中预设的测试 Key，当文件缺失时能抛出明确的自定义异常。

### Step 2: 构建 REPL 基础交互循环

- **指令**：引入 `prompt_toolkit` 构建一个持续监听输入的无限循环函数。配置支持多行输入与方向键历史记录的功能。实现优雅的异常捕获逻辑，专门处理 `KeyboardInterrupt`。
- **测试**：启动该交互循环，尝试输入普通文本并验证回显；尝试粘贴多段文本；按下 `Ctrl+C`，断言程序平滑打印退出提示并结束进程，而非抛出堆栈报错。

### Step 3: 接入多模型流式调用层

- **指令**：引入 `LiteLLM` 作为模型网关。编写一个底层模型调用函数，接收标准的消息列表。配置开启流式（Streaming）输出，并结合 `Rich` 库实现终端的打字机渲染效果。在此函数中实现手动接管的异常分类处理机制（处理限流、网络波动与致命错误）。
- **测试**：向该函数传入一条简单的问候语提示词，验证终端能够平滑、逐字地渲染模型回复。修改为一个无效的 API Key，断言程序能触发鉴权失败的致命错误机制并安全退出。

## 阶段二：工具层原子化抽象 (Milestone 2 铺垫)

将 Agent 与外界交互的"手脚"封装为纯净的、无状态的原子工具。

### Step 4: 实现 Python 原生执行器工具

- **指令**：使用 Python 的 `subprocess` 模块封装一个隔离的代码执行函数。该函数仅接收字符串格式的代码，将其写入临时文件或直接执行。配置参数以捕获标准的正确输出（stdout）和错误输出（stderr），并设置合理的执行超时限制。
- **测试**：分别进行两次断言测试。第一组传入打印字符串的正确代码，断言返回正确的 stdout；第二组传入含有故意拼写错误的代码，断言函数不崩溃，且精确返回包含错误堆栈信息的 stderr。

### Step 5: 实现 Web 搜索工具

- **指令**：集成 `tavily-python` 库，封装一个网络搜索工具函数。该函数接收查询字符串，执行调用后，负责过滤原始响应，仅返回提取好的相关上下文纯文本。
- **测试**：调用该函数查询当日的一个时效性新闻事件，断言返回的结果字符串中包含预期的关键词。

## 阶段三：工作流状态机编排 (Milestone 2 & 3)

利用图结构串联前面的组件，构建带有 ReAct 和反思机制的大脑。

### Step 6: 定义状态图结构与基础节点

- **指令**：使用 `LangGraph` 初始化状态流转的载体。定义包含消息历史的状态数据结构。构建最初始的图结构，包含一个专门负责调用已封装的 LiteLLM 函数的 "Agent 节点"，并将其与图的结束节点相连。
- **测试**：通过起始点传入一条测试消息，验证能够完整走通图的执行路径，并在终端获取最终的大模型回复。

### Step 7: 闭环 ReAct 逻辑与 Tool 节点集成

- **指令**：在图中新增一个处理工具调用的 "Tool 节点"，将先前封装的 Python 执行器和 Web 搜索工具注册其中。引入条件边（Conditional Edges）逻辑：Agent 节点执行后，解析其输出，若判定需要调用工具，则将状态路由至 Tool 节点，执行完毕后转回 Agent 节点；若无需调用，则结束。同时，使用 `Rich` 的状态功能，在路由至 Tool 节点时渲染终端加载动画（Spinner）。
- **测试**：输入指令要求计算一个大数字的乘积。观察终端，验证程序是否展示了工具调用的状态动画，断言状态机成功流转至 Python 执行节点，计算出结果后流转回 Agent 节点输出最终答案。

### Step 8: 实现错误反思与自愈循环 (Reflection)

- **指令**：修改 Tool 节点的处理逻辑。当执行 Python 工具检测到 stderr 有值时，将错误信息包装成一种特定类型的观察结果（Observation），连同失败的代码一并存入状态并传回 Agent 节点，要求 Agent 重新审视并修正代码。
- **测试**：输入指令强制要求 Agent "写一段含有除零错误的 Python 代码并运行"。监控状态机的流转，断言程序在第一次执行失败后，能够捕获报错内容，自动触发新一轮的 Agent 思考，修改代码为安全运算后重新执行并成功。

## 阶段四：产品化封装

将跑通的核心逻辑转化为开箱即用的命令行工具。

### Step 9: 全局 CLI 命令打包部署

- **指令**：引入 `Click` 或 `Typer` 库，将你的主 REPL 循环入口函数注册为一个命令行应用组的默认命令。编写 `setup.py` 或 `pyproject.toml`，配置控制台脚本（Console Scripts）入口，命名为你期望的全局命令字（如 `myagent`）。
- **测试**：在开发环境中执行本地可编辑安装（`pip install -e .`）。打开一个新的独立终端会话，直接键入该全局命令字，断言能够瞬间唤醒程序并进入熟悉的交互式 REPL 界面。
